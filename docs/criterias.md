# Итоговая оценка проекта по критериям

## Соответствие кода изученным принципам
- [] DRY (Don't repeat yourself)
- [] KISS (Keep it simple, stupid)
- [] SOLID
  - [] Принцип единственной ответсвтенности
  - [] Принцип открытости/закрытости
  - [] Принцип подстановки Лисков (использование подтипов базового типа, не зная об этом)
  - [] Принцип разделения интерфейсов (много интерфейсов лучше, чем один раздутый)
  - [] Принцип инверсии зависимостей (зависимость на абстракциях, не на реализации)
- [] Чистота кода: comments duplicate code, Magic Const, etc.
- [] Корректность инкапсуляции, абстракций, наследования

## Корректная, обоснованная реализация архитектуры

## Полноценность проекта

- [] Работоспособность (нерабочий проект не оценивается)

- [] Тесты
- [] Наличие стабильной гит-истории (отсчет с 1.04)
