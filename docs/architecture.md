# Архитектура проекта
## Глобальные решения
* EventBus
* Observer/Events модель
* MVP model для игровых объектов и глобальной архитектуры
  * MVP имеет единую точку входа, что упрощает взаимодействие и уменьшает раздробленность объекта, сохраняя плюсы архитектурной красоты.

## Game Loop Pipeline
Стандартный конвейер кадра.
1. INPUT  → Сбор клавиш/мыши → преобразование в команды
2. UPDATE → Обновление логики (физика, ИИ, таймеры, спавн, урон)
3. SYNC   → Копирование состояния Model → View (только если MVP)
4. DRAW   → Отрисовка всего на поверхности
5. FLIP   → Вывод на экран + ограничение FPS + расчёт dt для следующего кадра

## Сцены
Сцена - контейнер для текущего состояния мира, содержащий сущности и обновляющий это состояние в целом для всех.
Своеобразный синхронизатор.

Важно: чтобы сцена не стала GodObject, используем разделение ответсвтенности. Сцена только
вызывает системы в необходимом порядке, но не знает реализации.

## Системы типа физики и коллизии (разделение ответственности)
1. EntityManager отдаёт список [ICollidable]

2. PhysicsSystem.update(dt, collidables)
   1. только читает .get_collision_rect() и .get_collision_layer()
   2. при пересечении публикует EVENT_COLLISION_DETECTED
3. EventBus
   1. рассылает слушателям
4. PlayerPresenter / TrapSystem / DamageSystem
   1. реагируют на событие (меняют HP, отбрасывают, запускают анимацию)
