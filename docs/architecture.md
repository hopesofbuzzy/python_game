# Архитектура проекта
## Глобальные решения
* EventBus
* Observer/Events модель
* MVP model для игровых объектов и глобальной архитектуры
  * MVP имеет единую точку входа, что упрощает взаимодействие и уменьшает раздробленность объекта, сохраняя плюсы архитектурной красоты.

## Game Loop Pipeline
Стандартный конвейер кадра.
1. INPUT  → Сбор клавиш/мыши → преобразование в команды
2. UPDATE → Обновление логики (физика, ИИ, таймеры, спавн, урон)
3. SYNC   → Копирование состояния Model → View (только если MVP)
4. DRAW   → Отрисовка всего на поверхности
5. FLIP   → Вывод на экран + ограничение FPS + расчёт dt для следующего кадра
